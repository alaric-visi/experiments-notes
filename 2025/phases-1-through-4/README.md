# Phases 1 Through 4

## Overview
This project presents four independent mathematical visualisations — Hopf Fibration, Klein Bottle, Lorenz Attractor, and Mandelbulb — each rendered on its own 2D Canvas element. Only one visualisation is active at a time. A shared navigation system cancels the active `requestAnimationFrame` loop, fades the outgoing canvas to invisible, then activates and starts the incoming one. Each visualisation is initialised lazily on first selection and never re-initialised.

## Animation Switching
`switchAnimation` clears all active class markers, cancels all four `requestAnimationFrame` IDs, then after a 50 ms delay activates the target canvas and resumes its frame loop. The 50 ms gap allows the CSS `opacity: 0` transition to begin before the new canvas starts redrawing, preventing a frame of the old content appearing at full opacity in the new canvas. Each canvas has `position: fixed` and `opacity: 0` by default; the `active` class sets `opacity: 1` with an 0.8 s ease transition.

## Phase 1 — Hopf Fibration
The Hopf fibration maps circles from the 3-sphere S³ to points on S². `quaternionToS3(theta, phi, psi)` parameterises a point on S³ using spherical coordinates, deriving a unit quaternion `(a, b, c, d)`. `hopfMap` applies stereographic projection from S³ to ℝ³ by dividing the first three components by `1 − d`. Iterating `psi` from 0 to 2π at fixed `(theta, phi)` traces out one fibre — a circle in ℝ³.

A third of the fibres are generated using `generateVillarceauCircle`, which parameterises points on the surface of a torus in S³ using two angles `(u, v)` before applying the Hopf map, producing Villarceau circles on the projected torus.

Each frame, all fibres are rotated in ℝ³ via three sequential Euler angle rotations that advance with time and are offset by mouse position. A perspective projection divides by `depth = 4.5 + z` and scales by `baseScale × breathing`, where `breathing` is a compound sine/cosine function producing a slow scale pulse. Previous projected positions are stored per-point; each point is connected to its predecessor by a line segment, producing a continuous curve that trails across frames due to the semi-transparent fade overlay (`rgba(0,0,0,0.032)` per frame).

## Phase 2 — Klein Bottle
The Klein bottle surface is parameterised by `kleinBottle(u, v)`, which applies the standard piecewise immersion formula in ℝ³, using the `u < π` / `u ≥ π` split to handle the self-intersection region. The remaining curves are torus knots, generated by `torusKnot(t, p, q)` using integer winding numbers `p` and `q` chosen at random per curve.

Curves are rendered using the same trail and perspective system as Phase 1, with independent per-axis rotation speeds and mouse-driven offsets. A nebula particle layer of 360 pulsing coloured dots is rendered each frame before the curves using `source-over` compositing.

## Phase 3 — Lorenz Attractor
`LorenzAttractor` implements the Lorenz system using explicit Euler integration with a fixed sub-step of `dt / 2` per frame. Parameters `σ`, `ρ`, and `β` are individually perturbed from the canonical `(10, 28, 8/3)` values to produce distinct but qualitatively similar trajectories. Each attractor retains the last 700 positions as a history ring buffer via `Array.shift`.

The trail for each attractor is drawn segment by segment, with alpha set to `0.5 × age²` where `age = j / history.length`, causing older segments to be more transparent. The current z-value of each point shifts its hue relative to the attractor's base colour. A connection pass draws faint lines between each attractor and its two nearest neighbours when they are within 20 units of each other in ℝ³.

Mouse input drives `targetRotX` and `targetRotY`, which `rotX` and `rotY` follow with a 3% lerp, providing smooth pointer-influenced rotation alongside the auto-rotating `rotZ`.

## Phase 4 — Mandelbulb
The Mandelbulb is visualised using ray marching. `mandelbulbDistance` computes a distance estimate for a given point using the power-8 spherical iterative formula: at each iteration, the current radius, polar angle, and azimuthal angle are computed, the radius is raised to the power, and the angles are multiplied by the power before converting back to Cartesian coordinates and adding the seed point. The distance estimate is `0.5 × log(r) × r / dr`, where `dr` is the running derivative accumulator.

`julia3D` performs a related iteration using a fixed constant vector `juliaC` instead of the origin, returning the normalised escape iteration count.

`RayMarcher` instances are distributed across the unit sphere using a Fibonacci lattice (golden angle in `θ`, acos spacing in `φ`) to achieve near-uniform angular coverage. Each ray steps from origin `[-3, 0, 0]` toward its direction, taking steps proportional to 0.8 × the distance estimate. The path of sample positions is stored; the displayed effect renders each segment as a coloured line, with alpha and hue modulated by proximity to the surface.

Rays are re-marched in batches: the condition `Math.floor(T * 2) % 3 === 0` gates re-marching to every third phase (approximately every 1.5 seconds), and the `power` parameter is animated as `8 + sin(T × 0.5 + i × 0.1) × 1.5`, slowly morphing the surface between power-6.5 and power-9.5 Mandelbulbs.
