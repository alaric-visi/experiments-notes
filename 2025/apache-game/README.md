# Apache Game

A vertically-scrolling canyon-flight game rendered on an HTML5 Canvas, styled with a phosphor-green CRT aesthetic. The player pilots a pixelated helicopter sprite through a procedurally-generated canyon while engaging three enemy types and collecting power-ups.

## Overview

The game is desktop-only. On load, `navigator.userAgent` is tested against a mobile device regular expression; if matched, `body.classList.add('mobile-active')` hides the game container and displays a full-screen keyboard-requirement warning instead.

The layout consists of a left HUD panel, a central `<canvas>`, and a right HUD panel, arranged in a CSS Flexbox row. `resizeCanvas` reads the rendered widths of both panels via `getBoundingClientRect` and sets `canvas.width` to the remaining space. All game logic is guarded by `if (!isMobile)`, so no game objects or listeners are created on mobile.

## Core Mechanisms

### Canyon Generation

`initCanyon` pre-computes 200 canyon segment points into `game.canyon.leftWall` and `game.canyon.rightWall`. Each segment is generated by `addCanyonSegment`, which computes a lateral centre offset and a width-narrowing value from a sum of five and three superimposed sine/cosine waves respectively, each at a distinct frequency. This produces continuous, non-repeating canyon shapes:

```
variation = sin(tĂ—0.0234)Ă—70 + cos(tĂ—0.0431)Ă—55 + sin(tĂ—0.0627)Ă—40
          + cos(tĂ—0.0089)Ă—60 + sin(tĂ—0.0153)Ă—35

narrowing = sin(tĂ—0.0179)Ă—55 + cos(tĂ—0.0347)Ă—45 + sin(tĂ—0.0521)Ă—30
```

`updateCanyon` shifts every wall point downward by `game.scrollSpeed` each frame, removes points that have passed below the canvas, and appends new points at the top via `addCanyonSegment`. The canyon is effectively infinite.

### Scroll Speed Escalation

Every 900 frames (15 seconds at 60 fps), `scrollSpeed` is incremented by 0.2, capped at 4.5. This increases both the rendering speed of the canyon and the rate at which the player must react to narrowing passages.

### Wall Collision and Shield Drain

Each frame, the update loop finds the canyon segment index closest to the player's y-coordinate by iterating `leftWall` and minimising absolute y-distance. If the player's x-position is within 2 px of either wall bound, `game.shields` is decremented by 0.5 per frame. Shield depletion to zero or below triggers `loseLife`.

### Drawing the Canyon

`drawCanyon` renders every other wall point (stride of 2) as a single connected polyline in `#00ff00`. Horizontal cross-section lines are drawn at intervals of `floor(canvas.height / 40)`, connecting the left and right wall points at the same index. A background grid of 50% opacity green lines at `canvas.width / 20` spacing provides depth cues.

### Enemy Types

Three enemy types are spawned at random x-positions within the canyon:

- **Drone:** 2 HP, rendered as a filled rectangle with diagonal cross-lines.
- **UAV:** 3 HP, rendered as a filled hexagon computed from six equidistant polar coordinates.
- **Mine:** 1 HP, low vertical velocity (0.5 vs 1.5 for others), rendered as a filled circle with a central dot.

A health bar is drawn above each enemy, coloured green above 50% health, yellow above 20%, and red below.

### Force Fields

`spawnForceField` creates vertical obstacle barriers of width `min(40, canvas.width Ă— 0.1)` and height 80. Each force field has 6 HP. On each frame, vertical sine waves are drawn through the barrier at 8 px horizontal intervals, offset by `sin(pulse + i Ă— 0.2) Ă— 4`. The barrier pulses in opacity using `0.3 + sin(f.pulse) Ă— 0.2`. Bullets deal 1 HP of damage; homing missiles deal 2 HP.

### Homing Missiles

When `[X]` is pressed and `game.missiles > 0`, a missile is created that targets `game.enemies[0]`. Each frame, the vector from the missile to the target is normalised, scaled by 0.5, and added to the missile's velocity components. Velocity magnitude is capped at 8 px per frame. If the target is no longer present in the enemy array, the missile continues on its current trajectory.

### Power-ups

Three power-up types (fuel, shield, missile) spawn as rotating diamonds every 240 frames. Each item increases the respective resource by 30, 40, or 5 units respectively, capped at their maximum values. The diamond shape is a closed polygon with four vertices at the cardinal intercepts of the item's bounding box, rotated incrementally each frame.

### HUD

Both side panels are updated each frame by `updateHUD`. Shields and fuel are represented as vertical bar fills with `transition: height 0.3s ease`. Below 30%, the bar gradient switches from green to red and the numeric label colour changes correspondingly. The animated scan line (a 1 px horizontal element cycling from top to bottom over 4 seconds) is shown only during the `playing` state.

## Controls

| Key | Action |
|-----|--------|
| Z | Fire cannon |
| X | Launch homing missile |
| `<` / `,` | Move left |
| `>` / `.` | Move right |
| Space | Start / pause / resume |
| Escape | Pause |
